# Resume key points — template

Canonical list of the candidate's strongest experience signals. Used by `prepare` (CL paragraph 2) and `answer` (Q&A) so generated text references real achievements, not improvised ones.

Fill this file with facts only — verified numbers, titles, companies, dates. Do not include aspirational claims.

Suggested structure:

## Work experience (most recent first)

- Role, company, dates, top 3–5 outcomes with numbers.

## Education

- Degree, institution, year.

## Certifications

- Name, issuer, year.

## Proof-points worth pulling into narrative

- Group metrics that can headline a paragraph (e.g., "30% conversion rate lift from 40+ A/B tests", "API rebuild doubled traffic").

## Things to avoid framing as

- Any positioning lines the candidate has vetoed (e.g., "my product" vs. "advisory role", specific companies to never name, etc.).

The prepare / answer commands read this verbatim and cite from it. Keep it factual, short, and current.
